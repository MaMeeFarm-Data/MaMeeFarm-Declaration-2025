# MMFARM-POL-2025  
### MaMeeFarm™ Proof-of-Human-Work License  
**Version 1.0 — 2025 Declaration**

## 1. Purpose of This License
The MMFARM-POL-2025 License provides a framework for using, referencing, and extending  
the MaMeeFarm™ Proof-of-Human-Work system.  
It establishes guidelines for:
- transparent documentation  
- responsible reuse  
- data integrity  
- fair recognition of human effort  
- and open methodological study

This license is not a substitute for national law,  
but a structured protocol for responsible interaction with this project.

---

## 2. Scope of Coverage
This license applies to:
- documentation created within the MaMeeFarm™ repositories  
- daily Proof-of-Human-Work records  
- time-series data generated from real activities  
- metadata structures, verification files, and audit trails  
- system architecture diagrams and explanatory materials

It does not cover:
- personal identities  
- private information  
- commercial assets outside this repository  
- independent datasets created by other organizations

---

## 3. Principles of Use
Any reuse, research, or extension of this project must follow these principles:

### 3.1 Transparency  
Derived works must maintain clear attribution and a transparent record of modification.

### 3.2 Integrity  
Verification methods (e.g., timestamps, SHA-256, commit history) must remain intact  
whenever referencing this system’s methodology.

### 3.3 Non-Misrepresentation  
No user may present this system as endorsed by a government, institution, or standard body  
unless formally recognized by those entities.

### 3.4 Respect for Origin  
The origin of this dataset—real-world agricultural labor—must not be distorted or mischaracterized.

Under MMFARM-POL-2025, users may:
- study the project structure

- create analytical tools

## 5. Prohibited Activities
Users may not:
- claim ownership of MaMeeFarm™ data

- use the data for deceptive, harmful, or unlawful activities  
- create derivative works that imply institutional endorsement  
- redistribute altered materials without clear documentation

When referencing MaMeeFarm™ data, include:
- source repository link  
- commit hash (if applicable)  
- timestamp of retrieval  
- description of the context of use  

This helps maintain a consistent verification chain  
and preserves long-term research integrity.
